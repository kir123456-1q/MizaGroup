# 01 技术选型总览

## 1.1 项目技术目标

- 支撑 **第一人称格点迷宫 + 回合战斗** 的 DRPG 垂直切片  
- 策划可通过 **配置表** 调整层、怪、掉落、对话，无需改核心代码  
- 单机 Windows PC 优先，后续可扩 Steam 发行  
- 小团队可维护：模块清晰、依赖克制、文档与仓库结构固定  

## 1.2 技术栈总览

```
┌─────────────────────────────────────────┐
│  Presentation (UI / 迷宫视图 / 音频)      │
├─────────────────────────────────────────┤
│  Gameplay (探索 / 战斗 / 城镇 / 剧情)     │
├─────────────────────────────────────────┤
│  Domain (角色 / 物品 / 技能 / 迷宫格)    │
├─────────────────────────────────────────┤
│  Infrastructure (存档 / 配置 / 事件 / 日志)│
└─────────────────────────────────────────┘
```

| 层级 | 技术选型 |
|------|----------|
| 引擎 | Unity 2022.3 LTS（推荐稳定）或 Unity 6 |
| 脚本 | C# 10+，.NET Standard 2.1 |
| UI | uGUI + TextMeshPro |
| 输入 | Unity Input System 1.7+ |
| 序列化 | Newtonsoft.Json 或 Unity JsonUtility |
| 配置 | CSV（源） + ScriptableObject（运行时缓存） |
| 版本管理 | Git + Git LFS（美术资源） |
| IDE | Rider / Visual Studio 2022 |

## 1.3 已决策（ADR 摘要）

### ADR-001：采用 Unity 而非 Godot / 自研

| 选项 | 结论 |
|------|------|
| Unity | ✅ 选用：生态成熟、2D/3D 迷宫均可、招聘与教程多 |
| Godot | 备选：轻量但 DRPG 参考项目少 |
| RPG Maker | ❌ 第一人称 DRPG 不匹配 |

### ADR-002：迷宫视图方案

| 选项 | 结论 |
|------|------|
| 2D 预渲染背景切换 | ✅ **垂直切片推荐**：实现快、经典 Wizardry 观感 |
| 低模 3D 隧道 | 二期可选：层数多时资产复用高 |
| 纯 2D 俯视角 | ❌ 不符合 DRPG 第一人称定义 |

### ADR-003：战斗架构

| 选项 | 结论 |
|------|------|
| 自研轻量回合状态机 | ✅ 选用：需求可控 |
| Unity TurnBased 资产商店模板 | 可作原型参考，避免深度绑定 |

### ADR-004：配置数据源

| 选项 | 结论 |
|------|------|
| CSV + Editor 导入 | ✅ 策划友好 |
| 纯 ScriptableObject 手填 | 仅用于原型 |
| Excel 在线 + 自动同步 | 团队扩大后再上 |

## 1.4 非功能需求

| 项 | 目标 |
|----|------|
| 分辨率 | 1920×1080 窗口化；支持 1280×720 |
| 帧率 | 迷宫 60fps；战斗 UI 30fps+ 可接受 |
| 启动 | < 5s（SSD） |
| 存档 | 单档 < 500KB |
| 配置热更 | Editor 下可 Reload；发布版只读 |
| 平台 | Windows x64 首期 |

## 1.5 明确不采用（首期）

- 联机 / Netcode  
- Addressables 大型 DLC（垂直切片用 Resources 或本地 AssetBundle 即可）  
- ECS / DOTS（回合 RPG 收益低）  
- 可视化对话插件深度绑定（先 JSON/CSV 对话表）
