# 01 技术选型总览

## 1.1 项目技术目标

- 支撑 **第一人称格点迷宫 + 回合战斗 + 咏唱代价** 的 DRPG 垂直切片  
- 策划可通过 **配置表** 调整层、怪、掉落、对话、咏唱/模块，无需改核心代码  
- 单机 Windows PC 优先，后续可扩 Steam 发行  
- **个人开发者可维护**：Godot 轻量、迭代快、MIT 许可  

## 1.2 技术栈总览

```
┌─────────────────────────────────────────┐
│  Presentation (Control / 3D View / 音频) │
├─────────────────────────────────────────┤
│  Gameplay (探索 / 战斗 / 城镇 / 咏唱)    │
├─────────────────────────────────────────┤
│  Domain (角色 / 物品 / 技能 / 迷宫格)    │
├─────────────────────────────────────────┤
│  Infrastructure (存档 / 配置 / 信号 / 日志)│
└─────────────────────────────────────────┘
```

| 层级 | 技术选型 |
|------|----------|
| 引擎 | **Godot 4.4+**（GDScript） |
| UI | Control + Theme + 思源黑体 |
| 输入 | InputMap（键鼠 + 手柄） |
| 序列化 | JSON（`JSON.stringify` / `FileAccess`） |
| 配置 | CSV（源）→ Resource `.tres` 或运行时 Dictionary |
| 小地图 | TileMapDual 双网格插件 |
| 版本管理 | Git + Git LFS（美术） |
| IDE | VS Code + Godot Tools / 内置脚本编辑器 |

## 1.3 已决策（ADR 摘要）

### ADR-001：采用 Godot 4 而非 Unity / UE

| 选项 | 结论 |
|------|------|
| **Godot 4** | ✅ **选用**：个人开发迭代快、MIT 免费、格点 DRPG 模板多、TileMapDual 双网格 |
| Unity | 放弃：未推进且文档沉没成本可一次性迁移 |
| UE5 | ❌：2D/系统型 DRPG 过重 |
| RPG Maker | ❌：第一人称 DRPG 不匹配 |

### ADR-002：迷宫视图方案

| 选项 | 结论 |
|------|------|
| 2D 预渲染背景切换 | ✅ **垂直切片推荐** |
| 低模 3D 隧道（GridMap） | 二期可选 |
| 纯俯视角 | ❌ |

### ADR-003：战斗架构

| 选项 | 结论 |
|------|------|
| 自研 GDScript 回合状态机 + 咏唱读条 | ✅ |
| 第三方回合框架 | 仅参考，避免深度绑定 |

### ADR-004：配置数据源

| 选项 | 结论 |
|------|------|
| CSV + Editor `@tool` 导入 | ✅ |
| 纯手填 .tres | 仅原型 |
| Excel 在线同步 | 团队扩大后再上 |

### ADR-005：咏唱与整备（新增）

| 选项 | 结论 |
|------|------|
| 原著咒术代价 + SJ Main App 式模块 | ✅ 见策划案 11 |
| 纯 MP 技能无代价 | ❌ 丧失差异化 |

## 1.4 非功能需求

| 项 | 目标 |
|----|------|
| 分辨率 | 1920×1080 窗口；1280×720 可玩 |
| 帧率 | 迷宫 60fps |
| 启动 | < 3s（Godot 导出 SSD） |
| 存档 | 单档 < 500KB JSON |
| 配置热更 | 编辑器下 Reload；发布只读 |
| 平台 | Windows x64 首期 |

## 1.5 明确不采用（首期）

- 联机 / 多人  
- Godot .NET 版（除非后期有强需求）  
- ECS 架构  
- 全语音 / Addressables 式远程资源
