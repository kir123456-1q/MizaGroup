# 04 数据与配置方案

## 4.1 数据流

```
策划案/samples/*.csv
        ↓  Editor 「Miza → Import All Configs」(@tool)
data/generated/*.tres   或  启动时直读 CSV
        ↓  Boot Autoload
GameDatabase (Dictionary 索引)
        ↓
Gameplay / UI 只读
```

运行时进度 → `GameSession`（Autoload）→ `SaveManager` → `%APPDATA%/MizaDRPG/save_slot_0.json`。

## 4.2 Resource 设计

```gdscript
# scripts/domain/definitions/enemy_definition.gd
class_name EnemyDefinition
extends Resource

@export var id: String
@export var name_key: String
@export var rank: int
@export var hp: int
# ...
```

容器：

```gdscript
# enemy_database.gd
class_name EnemyDatabase
extends Resource

@export var enemies: Array[EnemyDefinition] = []

func build_index() -> Dictionary:
    var d := {}
    for e in enemies:
        d[e.id] = e
    return d
```

**原则**：`Definition` = 只读；`EnemyInstance` = 战斗临时 HP/Buff。

## 4.3 CSV 导入器（Editor @tool）

路径：`scripts/editor/csv_importer/`

1. 读取 `data/source_csv/` 或 `策划案/samples/`  
2. 解析 CSV（引号、逗号）  
3. 生成 `data/generated/enemy_database.tres`  
4. 校验：重复 id、悬空 `drop_id`  

菜单：`Miza → Import All Configs`、`Miza → Validate Configs`

可选 CI：

```bash
python tools/validate_configs.py DRPG方案/策划案/samples/
```

## 4.5 存档格式

路径：`user://save_slot_0.json`（Windows 映射到 `%APPDATA%/Godot/...` 或自定义 `OS.get_user_data_dir()`）

```json
{
  "version": 2,
  "timestamp": "2026-07-03T12:00:00",
  "session": {
    "party": [...],
    "inventory": { "items": [...] },
    "story_flags": ["met_lastiera", "tia_joined"],
    "installed_modules": ["mod_enemy_sense", "mod_safe_path"],
    "installed_chants": ["chant_wind_light"],
    "chant_debts": [{ "type": "mp_regen_floor", "floors_left": 1 }],
    "dungeon": {
      "current_floor": 3,
      "player_x": 5,
      "player_y": 8,
      "facing": "North"
    },
    "gold": 150
  }
}
```

`version: 2` 起含模块/刻印；`ISaveMigrator` 处理旧档。


## 4.9 「表示」UI 绑定

| UI 页 | 数据源 |
|-------|--------|
| 状态 | `GameSession.party[0]` + 公式 |
| 技能 | `SkillDatabase` + skill_levels |
| 持有物品 | `GameSession.inventory` |
| 注视 | 敌人 Def + 解析等级；`mod_poison_id` 显示毒种 |
| 刻印 | `installed_chants` + `chant_debts` |

## 4.10 实现顺序

1. `GameConst` → `LevelExp` → `SkillEffect`  
2. `Character` → `Skill` → `Item` → **`Chant`** → **`UpgradeModule`**  
3. `Enemy` → `DropTable` → `EncounterPool`  
4. `DungeonFloor` → `MapTile`  
5. `Shop` → `Dialogue` → `Quest` → `Localization`
